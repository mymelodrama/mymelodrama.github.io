2.部署到网站

​ 首先，你需要在自己的github上面上建立一个和自己用户名同名（建议小写）的的仓库😱（重点！由于github上发布你的博客的仓库必须和自己的用户名一样，否则会有bug加载不出来，这也是搭建博客时大部分的部署不上去的坑）

​ 下一步将我们的Hexo与GitHub关联起来，打开站点的配置文件_config.yml，翻到最后修改为：

deploy:
type: git
repo: 这里填入https://github.com+你的github用户名+你的博客对应仓库url(仓库名和自己的用户名同名但要小写)+git

🙂一定记得加上 .git（例如 https://github.com/mymelodrama/mymelodrama.github.io.git）
branch: master

ps：repo、branch、type等之后要有一空格 否则会出现格式错误

自己所遇到坑😟

1.本地图片的加载问题

如果本地图片导入后在页面上加载不出来有以下几种方法：

路径是否正确：

​ 打开localhost:4000生成的静态页面，F12查看图片源码的路径是否正确

红线所示的统一资源定位器(url)必须为博客对应的自己的github仓库，若不是请在站内配置文件中blog/_config.yml中 url项修改为

url:自己博客github仓库的url
root:/你的仓库名.+github.+io/(两边要有下划线)

增加本地相对路径图片插件

​ 确保你的Hexo的配置文件_config.yml里面有个这个选项配置，并将其置为true

生成一篇新文章时，会在\source_posts目录下生成一个 文章名.md 文件外，附带生成一个与 文章名 同名的文件夹，可以用它来存放这篇文章的所有资源，比如图片，附件等。

接下来你需要安装一个图片路径转换的插件，这个插件名字是hexo-asset-image

插件的安装及使用

2.图床

​ 图床，因为图片会占据大量的存储的空间，加载的时候相对缓慢 ，这时考虑把博文里的图片上传到某一网站，然后获得外部链接，使用Markdown语法，完成图片的插入，这种网站就被成为图床。

​ 亲测用的七牛云，效果不错，需要个人身份认证才能使用七牛云以及七牛云使用的简单教程

3.域名

​ 域名建议购买国内的，购买域名这也是我们整个搭建过程中惟一一个需要花钱的地方。看其他大佬的博客经济的选择是top域名，当然也可以用阿里云的域名，阿里云域名购买
